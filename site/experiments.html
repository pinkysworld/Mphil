Experiment Set

How the project tests performance, robustness, and reliability.

The experiment suite moves from simple classification baselines to stronger temporal and reliability checks. The aim is not only to produce a high score, but to understand when that score is trustworthy and when it begins to break down.

Baselines Temporal analysis Reliability

Baselines

Initial classification comparisons

Single-view baselines

These runs ask what each feature family can do on its own. API-hash and API TF-IDF baselines provide the main reference points.

Fused baselines

Fusion experiments combine several feature views to test whether mixed evidence is more useful than any individual representation.

Model comparison

SGD, logistic regression, and LightGBM are used as practical baselines, with particular attention to how they behave under chronological evaluation.

What the baseline runner does

It loads cached features for one selected view, applies a chosen split, trains the requested classifier, and writes the metrics, report tables, and confusion matrix outputs.

Why the baselines matter

They establish the first defensible result table. Every later analysis refers back to these runs when discussing drift, leakage, and calibration.

Temporal and Robustness Analysis

Tests that go beyond a single headline score

Invariance vs discriminability

This analysis asks whether a feature view is both predictive and reasonably stable across time. It is useful for separating strong but brittle signals from strong and durable ones.

Walk-forward evaluation

Training moves forward in time and each model is tested on the next monthly window. This reveals where performance is stable and where it drops under drift.

Leakage ablation

The ablation reruns the task with and without family-name filtering to see whether shortcut tokens are driving the result.

Reliability

Confidence, calibration, and abstention

Calibration analysis

Calibration examines whether a confidence score of, for example, 90% actually behaves like 90% accuracy in practice. This matters if the system is intended to support downstream decisions.

Selective prediction

Selective prediction asks whether the model becomes safer if it is allowed to defer on uncertain samples rather than always forcing a label.

Output set produced by the reliability script

  • Calibration metrics: summary JSON with uncalibrated and calibrated scores.
  • Reliability plots: diagrams for the confidence distribution before and after calibration.
  • Per-family ECE: a breakdown of confidence behaviour by predicted family.
  • Risk-coverage plot: comparison of max-probability and margin-based selection rules.
