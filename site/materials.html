Materials

Data handling, repository structure, and reproducibility notes.

The project is public at the code level, but the raw malware material and the larger generated outputs are intentionally kept local. This page records how that boundary is handled.

Data Repository Working notes

Data

What is used and what stays local

Used in the project

The current runs are based on the reduced Avast-CTU CAPEv2 dynamic-analysis release together with the public family labels.

Kept outside Git

Raw archives, extracted reports, processed caches, and generated experiment outputs are stored locally and excluded from version control.

Why the reduced release is practical

It preserves the label definitions needed for the core study while making ingestion, caching, and repeated experiments much easier to manage.

Why the raw archive is not pushed

The dataset is large, third-party, and not something that belongs inside a normal Git history. The repository is meant to hold the research code and notes, not the raw malware corpus.

Repository

How the project workspace is organised

site/                GitHub Pages website
scripts/             data pipeline and experiment scripts
configs/             configuration files
docs/                project notes and planning material
data/raw/            local raw archive area (ignored)
data/cache/          generated feature caches (ignored)
data/splits/         generated split files (ignored)
artifacts/           local experiment outputs (ignored)

The public repository is therefore a lightweight research workspace: code, notes, and the project website are visible, while the heavier data products remain local.

Working Notes

Current reproducibility approach

  • Environment: the project is run in a local Python virtual environment pinned to the required scientific stack.
  • Dataset handling: the ingestion script unpacks the archive automatically and matches it to the public label file.
  • Intermediate products: feature caches and experiment outputs are regenerated locally rather than tracked in Git.
  • Website: the site is deployed with GitHub Pages and is intended to serve as a concise project record alongside the thesis writing.

Hosted with GitHub Pages. Code and notes are public; raw dataset files and generated caches remain local.
